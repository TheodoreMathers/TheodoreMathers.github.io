C2 ==PE文件==

PE文件格式总体结构

  • DOS MZ header
    定位PE头文件开始位置（最后四个字节0，也可以用于PE文件合法性检测
  • DOS stub
    DOS下运行该程序时，将提示用户：“This Program cannot be run in DOS mode”
  • PE header
    • 字串"PE\0\0"(Signature)(50\45\0\0)
    • 映像文件头(File Header)包含PE文件物理分布信息，节数、后续可选文件头大小、机器类型等
    • 可选文件头(Optional Header)定义了PE文件的很多关键信息，内存镜像加载地址(ImageBase)、程序入口点、节在文件和内存中的对齐粒度、本程序在内存中的镜像大小，文件头大小等
  • Section table

为何需要导出序号表

导出函数名字和导出地址表中的地址不是一一对应关系

一个函数实现可能有多个名字

某些函数没有名字，仅通过序号导出

C3 软件缺陷与漏洞机理概述

什么是软件漏洞

漏洞是存在于评估对象中的、在一定环境条件下可能违反安全功能要求的弱点

漏洞是存在于信息系统，系统安全过程，内部控制或实现过程中的、可被威胁源攻击或触发的弱点

漏洞是能够被一个或多个威胁利用的资产或控制中的弱点

漏洞是贯穿软件生命周期各环节的。在时间维度上，漏洞都会经历产生、发现、公开、消亡等过程

为什么会出现软件漏洞

计算机系统结构决定了漏洞的必然性————

软件趋向大型化，第三方扩展增多

新技术新应用产生之初即缺乏安全性考虑

软件使用场景更具威胁

对软件安全开发重视不够，软件开发者缺乏安全知识

软件漏洞利用对系统的威胁

非法获取访问权限

权限提升

拒绝服务

恶意软件植入

数据丢失或泄露

漏洞的分类

通常可以从漏洞利用的成因、利用的位置、和对系统造成的直接威胁进行分类

  • 基于漏洞成因的分类

    • 输入验证类

  • 基于威胁类型的分类

C4 缓冲区溢出漏洞

堆栈溢出示意

当写入内存的数据大于我们分配的长度，导致邻近的内存数据被覆盖，如果精心准备覆盖到程序返回指针的数据，程序的进程可能就会被攻击者所控制

缓冲区溢出的危害

成功地利用缓冲区溢出漏洞可以修改内存中变量的值，甚至可以劫持进程，执行恶意代码，最终获得主机的控制权

要透彻的理解这种攻击方式，需要计算机体系架构方面的基础知识，理解CPU、寄存器、内存是怎样协同工作而让程序流畅执行的

栈溢出的利用

修改邻接变量

修改函数返回地址

S.E.H.结构覆盖

修改邻接变量

由于函数的局部变量是依次存储在栈帧上的，因此如果这些局部变量中有数组之类的缓冲区，并且程序中存在数组越界缺陷，那么数组越界后就有可能破坏栈中相邻变量的值，甚至破坏栈帧中所保存的EBP、返回地址等重要数据

修改函数返回地址

通过覆盖修改返回地址，使其指向shellcode地址，更改程序流程，从而转至shellcode处执行

堆的结构

空闲堆块比战友堆块多出了两个4字节的指针，这两个指针用于连接系统中的其他空闲块

空表
堆表区中有一个128项的数组，称做空表索引（每项包含两个指针，标示一条空表）
空闲堆块的大小=索引项 x 8（字节）

堆溢出的利用

当精心构造的数据淹没前向指针和后向指针时，如果在flink放入4字节的任意恶意数据内容，在blink放入目标地址，当执行以上操作时，导致目标地址的内容被修改为4字节的恶意数据

C4 Web基础知识

目前普遍使用的Web三成架构

典型的Web安全漏洞

Web应用安全漏洞是Web应用程序在需求、设计、实现、配置、维护和使用等过程中，有意或无意产生的缺陷，这些缺陷一旦被攻击者所利用，就会造成对网站或用户的安全损害，从而影响构建于Web应用之上的正常服务的运行，危害网站或用户的安全属性

C5 软件漏洞利用和发现

漏洞利用简介

Exploit
可以根据目标环境构造一段用于实现攻击的二进制串（Payload）

Payload
被注入到目标进程触发漏洞获得执行权限的二进制串
代表攻击者攻击意图的代码（Shellcode）

恶意代码的定义与发作趋势

设计目的是用来实现恶意功能的代码或程序

正常的软件也会引发安全问题，但绝大多数情况并非作者有意

恶意代码的分类

计算机病毒、蠕虫
计算机病毒，一组能够进行自我传播，需要用户干预来出发执行的破坏性程序或代码
蠕虫，一组能够进行自我传播、不需要用户干预即可触发执行的破坏性程序或代码

木马、后门
木马，看起来具有正常功能，但实际上隐藏着很多用户不希望功能的程序。通常由控制端和被控制端组成
后门，是的攻击者可以对系统进行非授权访问的一类程序

Rootkit
通过修改现有的操作系统软件，使攻击者获得访问权并隐藏在计算机中的程序

僵尸（bot）

流氓软件、间谍软件

广告软件、Exploit、黑客工具

C10 恶意软件防范技术

恶意代码的检测

将检测对象与恶意代码特征（检测标准）进行对比分析，定位病毒程序或代码，或检测恶意行为

检测对象

引导扇区

文件系统中可能带毒的文件

内存空间

主板BIOS等

特征值检测技术

病毒特征值是反病毒软件鉴定特定计算机病毒的一种标志。通常是从病毒样本中提取的一段或多段字符串或者二进制串
获取样本->提取样本特征=>更新病毒库->查杀病毒

特征值的提取选择

特定字串

感染标记

从病毒代码特定的地方开始取出的连续的、大于64且不含空格的（ASCII值为32的字节串）

提取方法

人工提取
反病毒工程师对病毒样本进行分析后，人工确定病毒特征

自动提取
通过软件系统自动提取特定范围内特定长度具有一定特征的数据
处理不力则可能被别有用心者利用，形成误杀

对抗

手工修改自身特征
首先，利用反病毒软件定位
然后，进行针对性修改

自动修改自身特征
加密、多态、变形等

校验和检测技术

在文件使用/系统启动过程中，检查检测对象的实际校验和预期是否一致，因而可以发现文件/引导区是否感染

预期

正常文件内容和正常引导扇区数据

校验和检测对象

文件头部

文件属性

文件内容

系统数据等

C11 恶意软件捕获方法

蜜罐

通常指未采取安全防范措施、并且将模拟的程序漏洞主动暴露在网络中的计算机

特点
与一般的计算机不同，其内部运行着多种多样特殊用途的“自我暴露程序”和行为记录程序
引诱恶意软件在蜜罐内更加充分的运行，并记录下其行为

工作模式
被动型蜜罐
主动型蜜罐

主动型蜜罐实现思路
S1:通过爬虫等主动获取潜在的恶意软件
S2:将其在蜜罐主机内打开、运行，并模拟进行交互
S3:根据运行特征，发现和收集漏洞利用信息和恶意软件样本
