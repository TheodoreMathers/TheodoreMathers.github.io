C1 计算机系统概述

1.2.1 计算机系统的组成

📖冯诺依曼计算机体系结构的基本思想

存储程序。将指令以代码的形式事先输入计算机的主存储器，然后按其在存储器中的首地址执行程序的第一条指令，以后就按该程序的规定顺序执行其他指令，直至程序执行结束。

1.3.1 计算机的主要性能指标

📖机器字长

计算机进行一次整数运算所能处理的二进制数据的位数，机器字长一般等于内部寄存器的大小，它决定了计算机的运算精度。

📖指令字长

一条指令的二进制代码的位数。

存储字长

一个存储单位能存储的二进制代码的长度。

📖指令字长一般是存储字长的整数倍。现代 CPU 的机器字长和指令字长不一定相等，但两者都是字节的整数倍。

📖机器周期

所有指令过程中的一个基准时间。

📖指令周期

CPU 取出并执行一条指令所需的全部时间。

📖一般情况下，一个指令周期包含若干个机器周期，每个指令周期内的机器周期数可以不一样。

C3 存储系统

3.1 存储器概述

3.1.1 存储器的分类

Cache

Cache 是一种 SRAM 存储器，是为了解决 CPU 和主存之间速度不匹配而采用的一项重要硬件技术。现发展为 Cache-主存体系。

闪速存储器

闪速存储器能提供高性能、低功耗、高可靠性以及可擦写能力，为现有的存储器结构带来巨大变化，因此作为固态硬盘用于便携式电脑中。

3.1.2 存储器的性能指标

速度排列

寄存器 > Cache > RAM > ROM

📖存取周期

存储器连续两次独立地读或写操作所需的最小时间间隔。

📖存取时间

读出时间：主存接收到有效地址到数据稳定为止的时间。写入时间：主存接收到有效地址到数据被写入单元为止的时间。

📖存储周期包括存取时间和恢复时间，通常存取周期大于存取时间。

3.3.1 SRAM 和 DRAM

SRAM 引线数目

通常情况：地址位 + 数据位 + 1 个片选位 + 1 个读写控制位+（通常题目不考虑的：电源、地线）

DRAM 刷新方式

全局刷新，分散刷新，异步刷新。

3.6.3 Cache 和主存的映射方式

方式分类

直接映射、全相联映射、组相连映射

🧮组相连映射

主存地址字段：主存字块标记:组地址:组内地址

命中路的计算

C4 指令系统

4.1.1 指令的基本格式

指令内容

  1. 操作码，定义区分不同指令。
  2. 寻址特征，区分不同的寻址方式。
  3. 地址：地址数量决定指令性质。
  4. 相对地址（位移量）

指令里面的形式地址算是一个地址字段吗？王道里面描述直接寻址的时候，说直接寻址是形式地址存操作数的真实地址，这种是算成了一个地址字段吧；西工大期末题里面，题目A：一个指令有两个寄存器字段（一个源寄存器一个基址寄存器）和一个偏移量字段，叫二地址指令。题目B：一个指令有两个寄存器字段（一个源寄存器一个变址寄存器）和一个形式地址字段，叫可为二地址指令。写到这里好像懂了，形式地址里面存形式地址（偏移量）的时候不算一个地址字段，形式地址字段存地址的时候算一个地址字段。总结：形式地址字段既可以存偏移量又可以存地址，算不算一个地址字段靠具体存的东西判断。

4.1.4 指令的操作类型

数据传输、算数运算、逻辑运算、数据传输、程序控制、I/O、字符串、系统控制

4.2.2 常见的数据寻址方式

寻址方式

  1. 隐含寻址
    不明显给出操作数地址，不一定没有地址（比如一个指令需要两个操作时，给出一个，隐含另一个）通常将 ACC（累加器）作为隐含地址。
  2. 立即寻址
    地址字段存放的不是操作数的地址，而是操作数本身，以补码的形式存放。指令执行阶段不访问主存，执行时间最短。位数限制了立即数的范围。
  3. 直接寻址
    指令中的地址字段即为操作数在主存中的地址。在指令执行阶段仅访问一次主存。位数限制了寻址范围，操作数的地址不易修改。
  4. 间接寻址
    地址字段存放操作数地址的地址。主存单元第一位作为标识，若是 1 则说明取出来的仍是地址，继续寻址，若是 0 则表示找到了操作数。（访问速度过慢，虽然可以扩大寻址范围，但一般使用的是寄存器间接寻址）
  5. 寄存器寻址
    指令中的地址字段给出操作数所在的寄存器编号。在指令执行阶段不访问主存，只访问寄存器，支持向量/矩阵运算。寄存器价格昂贵，个数有限。
  6. 寄存器间接寻址
    指令中的地址字段给出操作数地址所在的寄存器编号。比一般间接寻址快，但仍需要访问主存。（操作数在主存）
  7. 相对寻址
    程序计数器加上形式地址（偏移量）形成操作数的有效地址。广泛用于转移指令。
  8. 📖基址寻址
    指令中含有基址寄存器字段。基址寄存器加上形式地址（偏移量）得到实际地址。可扩大寻址范围，但偏移量较短。
  9. 📖变址寻址
    指令中含有变址寄存器字段。变址寄存器加上形式地址（偏移量）得到实际地址。适合数组、循环程序。（📖基址寻址通常用作操作系统为多道程序分配存储空间，基址寄存器的值在寻址时不变；变址寻址立足于用户，主要用于处理数组问题，变址寄存器的值在寻址时会变）
  10. 堆栈寻址
    无地址字段，有一块特定的存储栈区，先进先出。

C5 中央处理器

5.1 CPU 的功能和基本结构

5.1.1 CPU 的功能

（操作）控制器的功能

协调并控制计算机各部件执行程序的指令序列，包括取指令、分析指令和执行指令

5.1.2 CPU 的基本结构

时序系统

用于产生各种时序信号，他们都由统一时钟（CLOCK）分频得到

CPU 结构

5.2.2 指令周期的数据流

📖CPU 如何区分指令和数据

从时间上讲，取指令事件发生在“取指周期”，去数据事件发生在“执行周期”。从空间上讲，从 PC 提供的存储单元地址取出的是指令，从指令地址码部分提供的存储单元地址取出的是操作数。

数据流的描述

寄存器向存储器读写数据时，在存储器旁括号写出地址来源，如：

MDR—>M(MAR)，根据 MAR 提供的地址，将 MDR 中的数据写入 M。

5.4.3 微程序控制器

机器指令与微指令的关系

许多条微指令组成的序列构成微程序，微程序完成对指令的解释执行。一条指令对应一个微程序，一个微程序由许多条微指令构成，一条微指令会发出很多不同的微命令。

时序体制

在微程序控制器中，一般采用较简单的节拍电位、节拍脉冲二级时序体制。

水平型微指令格式

字段直接编码：按题目给的分组，每一组加1种全0不操作状态进行编码，再加上可判断的外部状态的判断测试字段，剩下的就是后继地址字段。

5.5.1 指令流水线的基本概念

流水线时钟周期的选择

以所花时间最长的操作部分为主。

相邻指令数据相关

IF ID EX WB
IF ID EX WB

当相邻指令发生数据相关时，第二条指令必须等第一条指令将运行结果回写结束之后才能读取操作数，因此必须往后延迟两个流水线时钟。

否则可以采用数据旁路技术，第一条指令EX的结果直接作为第二条指令EX的操作数。

C6 总线

6.1.1 总线基本概念

📖 总线的主设备与从设备

主设备：获得总线控制权的设备

从设备：备注设备访问的设备，它只能响应从主设备发出的各种总线命令

6.1.3 总线系统的结构

总线结构速度的比较

多总线系统 > 三总线系统 > 双总线系统 > 单总线系统

6.2.1 集中仲裁方式

📖 为什么要仲裁

解决多个主设备同时竞争总线控制权的问题。

进行总线仲裁的时机

当总线空闲时才进行仲裁，总线忙直接返回忙信号。

独立请求

缺点：控制线数量多（不会增加仲裁部件的负载）

6.3 总线操作和定时

总线定时

总线双方交换数据的过程中需要时间上配合关系的控制，这种控制称为总线定时，其实质是一种协议或规则，主要有同步和异步两种基本定时方式。

6.3.1 总线传输的 4 个阶段

  1. 申请分配
  2. 寻址
  3. 传输
  4. 结束

6.3.2 同步定时方式

📖 同步通信

由统一时钟控制，总线周期的长度固定，控制方式简单，灵活性差。当系统中各部件工作速度差异较大时，总线工作效率明显下降。适合于速度差别不大的场合。具有高的传输频率。缺点：同步总线必须按最慢的模块来设计公共时钟，当模块存取时间相差很大时，回答大损失总线效率。

为什么速度快

不需要应答信号、总线长度短（各部件距离近）、用一个公共时钟信号进行同步、各部件存储时间接近

6.3.3 异步定时方式

异步通信

没有统一时钟控制，部件间采用应答方式进行联系，控制方式较同步复杂，灵活性高。当系统中各部件工作速度差异较大时，有利于提高总线工作效率。优点：总线周期长度可变，不把响应时间强加到功能模块上，因为允许快速和慢速的功能模块都连接到同一总线上。缺点：总线复杂、成本较高。

C7 输入/输出系统

7.1.1 输入/输出系统

通道

通道与 CPU 分时使用主存，实现了 CPU 内部数据处理和 I/O 设备的并行工作。当通道完成工作后，向 CPU 发送中断请求。

7.3.1 I/O 接口的功能

接口的功能

地址译码、设备选择、传送控制信息状态信息

7.3.2 I/O 接口的基本结构

I/O 指令的功能

选址、传送命令、实现数据交换

I/O 指令不实现 I/O 数据传送，主要完成启、停 I/O 设备，查询通道和 I/O 设备的状态，及控制通道进行其他一些操作等

7.3.4 I/O 端口及其变址

编址方式与访问方式

统一编址：把 I/O 端口当作存储器的单元进行地址分配，使用统一的访存指令就可以访问 I/O 端口。I/O 端口需固定在主存的某一部分

独立编制：无法从地址码上分辨主存地址和 I/O 端口的地址，需使用专门的 I/O 指令来访问 I/O 端口

7.4.2 程序中断方式

关中断

在关中断状态时，CPU 不会响应外中断（可屏蔽中断），内中断（异常、不屏蔽中断）正常响应。

程序中断方式工作流程

  1. 中断请求
  2. 中断判优
  3. （判断是否响应）中断响应
  4. 中断向量
  5. 中断处理过程
