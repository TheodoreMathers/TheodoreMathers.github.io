C1 概述

1、==Java EE==

  • Java EE 是 Java 企业版，针对企业级应用
  • Java EE 已成为企业级开发的首选平台之一
  • Java EE 是一系列的技术规范，而并非是一个产品，更不是编程语言

2、==Java Web 开发所涉及的技术==

Intellij IDEA

集成开发环境

Maven

Maven 是一种基于项目对象模型(POM Project Object Model)，通过配置文件(pom.xml)来管理项目构建的一种软件项目管理工具

Thymeleaf

网页模板引擎

Postman

Postman 是一个 http 请求模拟工具，常用于测试后台接口

MySQL

数据库管理系统

JPA

JPA (Java Persistence API) 是一种 Java 持久化 API，用于对象的持久化，是一种非常强大的 ORM (Object Relation Mapping) 解决方案

3、==MVC 开发模式==

MVC 模式是 Web 开发中最常用的分层开发模式，即模型-视图-控制器模式

模型 model：应用的实体类，用于在内存中暂时存储数据，并在数据变化时通知控制器

视图 view：主要用来解析、处理、显示内容，并进行模版的渲染

控制器 controller：处理浏览器的请求，决定如何调用业务层的数据增删改查等业务操作，以及如何将结果返回给视图进行渲染

4、==Spring MVC 模式基本体系结构==

5、==Maven==

Maven 是一种基于项目对象模型(POM Project Object Model)，通过配置文件(pom.xml)来管理项目构建的软件项目管理工具

6、==新建 Spring Boot 项目时选择的三个 starter 组件的作用==

Spring Boot Devtools

用于项目热部署（为了更好支持程序调试，在项目进行修改之后不需要耗费时间重启，在程序正运行情况下即可实时生效。以节约时间和操作，提高开发效率）

Spring Web

用于构建Web应用程序

Thymeleaf

网页模板引擎

7、==Spring Starter==

Spring Boot 为了简化配置，提供了非常多的 starter，它打包好与常用模块相关的所有 jar 包，并完成自动配置，这使得开发时不需要过多关注框架配置，只需关注业务逻辑即可

9、常用注解

C2 Thymeleaf 基本用法

Thymeleaf 是一个服务器端 Java 模版引擎，适用于 Web 和独立环境，能够处理 HTML，XML，JavaScript，CSS 甚至纯文本等。Thymeleaf 主要目标是为开发工作流程带来优雅的自然模版。

@{ }

通过链接表达式 @{ } 直接拿到应用路径，然后拼接静态资源路径

C4 Spring Boot 控制层

1、==参数传递方法==

选择判断程序填空

无注解获取参数

要求：方法的参数名称要和 HTTP 请求的参数名称保持一致

特点：直接把 HTTP 请求的参数写在后台方法的形参中，允许参数为空

适用于 GET 和 POST 请求方式

@RequestParam 获取参数

使用 @RequestParam 注解来确定前后端参数名称的映射关系

适用于 GET 和 POST 请求方式

@RequestParam 有三个配置参数：

  • value 为接收 HTTP 请求的参数名
  • required 表示是否必须，默认为 true，表示参数必填
  • defaultValue 可设置请求参数的默认值

@PathVariable 获取参数

路径上必须有参数(required=true)，且不能设置默认值

适用于 GET 和 POST 请求方式

PathVariable(“xxx”) 将占位符参数"xxx"绑定到控制器方法的形参中

后台请求映射: @RequestMapping(“/test4/{id}/{name}”)，{xxx} 称为占位符参数

@RequestBody 获取 json

必须 POST 方式提交且必须与 contentType = “application / json” 配合使用

@DataTimeFormat

获取格式化日期参数

2、==Filter==

Filter 即过滤器，它是 Servlet 技术中最实用的技术，Web 开发人员通过 Filter 技术，对 Web 服务器管理的所有 Web 资源进行拦截，从而实现一些特殊的功能

3、==Interceptor==

Interceptor 即拦截器，它提供了一种机制可以使开发者在一个 Action 执行的前后执行一段代码，也可以在一个 Action 执行前阻止其执行。基于 Java 的反射机制，是 AOP 的一种运用。

4、==过滤器与拦截器的区别==

  • 拦截器是基于 Java 反射机制来实现的，而过滤器是基于函数回调来实现的
  • 拦截器不依赖 servlet 容器，过滤器依赖于 servlet 容器
  • 拦截器只对 Action 起作用，过滤器可以对所有请求起作用
  • 拦截器可以访问 Action 上下文和值栈中的对象，过滤器不能
  • 在 Action 的生命周期中，拦截器可以多次调用，而过滤器只能在容器初始化时调用一次

5、==session==

session 是一种记录服务器和客户端会话的机制。主要用来存储所有访问过该服务端的客户端的用户信息，从而实现保持用户会话状态

6、==token==

token 是服务端生成的一串字符串，作为客户端进行请求的一个令牌。当第一次登录后，服务器生成一个 token 便将其返回给客户端，以后客户端只需带上这个 token 前来请求数据即可，无需再带上用户名和密码

C6 JPA 技术实践

1、==如何理解 Entity 层、DAO 层和 Controller 层==

Entity 层：实体层，也就是所谓的 Model，是数据库在项目中的类，该文件包含实体类的属性和对应属性的 set、get 方法

DAO 层：持久层，主要负责与数据库进行交互，如 CRUD 等操作。DAO 层会调用 Entity 层

Service 层：主要负责业务模块的逻辑应用设计。一般先设计接口，再创建实现该接口的类。Service 层调用 DAO 层，接收 DAO 层返回的数据

Controller 层：控制层，负责具体的业务模块流程的控制。负责前后端交互，接受前端请求，调用 Service 层，接收 Service 层返回的数据，最后返回具体的页面和数据到客户端

2、==编程题==

根据本讲的示例，仿写：针对一个给出的实体类，写出相应的 Repository 仓库、服务层接口和实现（要求基本的 CRUD 功能），不写 Controller 层。注意：服务实现类上 @Service 不要掉了

2、==JPQL==

JPQL：全称 Java Persistence Query Language，是一种和 SQL 非常类似的对象查询语言，它最终会被编译成针对不同底层数据库的 SQL 查询，从而屏蔽不同数据库的差异

JPQL 支持 select、update、delete，不支持 insert
