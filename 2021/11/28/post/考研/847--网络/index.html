1.2.3 ISO/OSI 参考模型和 TCP/IP 模型

ISO/OSI

应用层
为网络应用提供协议支持和服务
表示层
协商应用程序间交互的数据格式
会话层
为进程之间的会话提供建立/维护/终止连接的功能
传输层
提供面向进程，面向连接或无连接的数据传输服务
网络层
提供路由选择、拥塞控制及网络互联功能、为端到端提供面向连接或无连接的数据传输服务
数据链路层
将不可靠的物理链路变成可靠的数据链路
物理层
完成 0/1 在物理介质上的传输

C3 数据路层

3.5 介质访问控制

3.5.2 随机访问介质访问控制

CSMA/CD

载波监听多路访问/碰撞检测

  • 征用型介质访问控制协议
  • LLC 帧封装在 Mac 帧的数据单元中
  • 适用于总线型网络

3.5.3 令牌传递协议

令牌传递协议

  • 只有一个令牌
  • 接受节点把数据取下后，还需要复制一份再发送到环路中
  • 采用确定性的介质访问控制协议，一般可用于对实时系统进行控制的环境
  • 需要对令牌进行维护和无源数据帧的维护

3.6 局域网

3.6.1 局域网的基本概念和体系结构

LLC 子层

逻辑链路控制

  • 识别网络层协议，对数据进行封装，为网络层提供服务。无确认无连接；面向连接；有确认无连接；高速传送
  • 在局域网体系中，数据链路层功能由 LLC 和 MAC 子层实现，LLC 封装在 MAC 帧中，因此没有校验或同步标志
  • LLC 帧地址是服务访问地址，不是物理地址，物理地址在 MAC 中
  • LLC 只定义了一种数据传输模式，简化了 DHLC

mac 子层

屏蔽了不同物理链路种类的差异性

vlan

  • 可以划分冲突域和广播域
  • 基于端口/ mac 地址/ IP 地址/应用划分

C4 网络层

4.3 IPv4

4.3.3 子网划分与子网掩码、CIDR

📖 子网掩码

屏蔽 IP 地址中的主机号，而保留网络号与子网号。用于说明 IP 地址中子网的位置。

4.3.4 ARP、DHCP 与 ICMP

ICMP Internet Control Message Protocol

网际控制报文协议

  • 终点不可达，（路径中的路由器向源点发送）
  • 源点抑制，（路径中的路由器要源点发慢点）
  • 时间超过，（TTL 为 0）
  • 参数问题，（首部出错）
  • 改变路由，（告诉主机新的更好的路由路径）

4.4 IPv6

4.4.2 IPv6 地址

IPv6

单播、多播、任拨（目的是一组计算机，但交付时只交付给一台）

C5 传输层

5.3 TCP 协议

5.3.3 TCP 连接管理

📖 TCP 建立连接

  1. SYN = 1；seq = x
  2. SYN = 1；ACK = 1；seq = y；ack = x + 1
  3. ACK = 1；seq = x + 1；ack = y + 1

📖 TCP 释放连接

  1. FIN = 1；seq = u
  2. ACK =1；seq = v；ack = u + 1
  3. FIN = 1；ACK = 1；seq = w；ack = u + 1
  4. ACK = 1；seq = u + 1；ack = w + 1
